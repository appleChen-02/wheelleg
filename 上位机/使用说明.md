# 上位机 USB 采集与 MATLAB 读取说明

## 1. 用途

本目录用于采集下位机通过 USB CDC 发送的数据，并将结果保存为 MATLAB 可读取的文件。

当前提供两个脚本：

- `usb_data_logger.py`：运行时接收 USB 数据并落盘为 CSV，必要时可导出 MAT。
- `read_usb_log.m`：在 MATLAB 中读取采集结果，返回结构体。

## 2. 依赖

Python 侧：

- Python 3.9 或更高版本
- `pyserial`
- 可选：`numpy`、`scipy`，用于导出 MAT 文件

MATLAB 侧：

- MATLAB R2019b 或更高版本更稳妥

## 3. 采集流程

1. 先确认下位机已经开启 USB 通信。
2. 在本目录运行 Python 采集脚本。
3. 采集停止后，目录里会生成 CSV 文件，必要时还有 `usb_log.mat`。
4. 在 MATLAB 中调用 `read_usb_log` 读取数据。

## 4. Python 采集脚本用法

在命令行进入本目录后执行：

```bash
python3 usb_data_logger.py --port /dev/ttyACM0 --baud 115200 --outdir ./log
```

也可以先列出串口：

```bash
python3 usb_data_logger.py --list-ports
```

如果不传 `--port`，脚本会自动选择一个串口（优先 `/dev/ttyACM*`、`/dev/ttyUSB*`）。

常用参数：

- `--port`：串口号，例如 `/dev/ttyACM0`（Linux）或 `COM6`（Windows）
- `--list-ports`：列出本机可用串口并退出
- `--baud`：波特率，默认 `115200`
- `--outdir`：输出目录，默认 `usb_log`
- `--duration`：运行多少秒后自动停止，`0` 表示一直运行
- `--save-mat`：退出时导出 `usb_log.mat`

示例：

```bash
python3 usb_data_logger.py --port /dev/ttyACM0 --outdir ./log_01 --duration 120 --save-mat
```

Ubuntu 若报权限错误（Permission denied）：

```bash
sudo usermod -a -G dialout $USER
```

重新登录后生效。

## 5. 生成文件说明

采集后会生成这些文件：

- `imu.csv`
- `robot_motion.csv`
- `unknown_frames.csv`
- `usb_log.mat`（如果启用 `--save-mat`）

其中：

- `imu.csv`：IMU 姿态与角速度
- `robot_motion.csv`：底盘速度矢量
- `unknown_frames.csv`：未识别的数据帧，便于调试协议

当前脚本按下位机 `usb_task` 的上行数据包解析：

- `0x02`：IMU
- `0x08`：RobotMotion

## 6. MATLAB 读取方法

进入 MATLAB 后执行：

```matlab
log = read_usb_log('C:\path\to\log');
```

如果当前 MATLAB 工作目录就是日志目录，也可以直接：

```matlab
log = read_usb_log(pwd);
```

读取结果说明：

- `log.imu`：IMU 数据表
- `log.robot_motion`：底盘速度表
- `log.mat`：如果存在 `usb_log.mat`，则加载后的结构体

## 7. 备注

- Python 脚本按下位机协议解析帧头和 CRC，直接可与固件对接。
- MATLAB 脚本优先读取 CSV，兼容性最好。
- 如果你后续修改了下位机数据包格式，只需要同步更新 Python 的解包格式和 MATLAB 读表字段即可。
